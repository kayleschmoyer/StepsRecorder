# Deployment

## Deployment Goal

The app should build into a professional Windows installer that can be installed by normal users.

## Target Platform

- Windows 11
- Windows 10 where technically feasible

## Build System

Use Tauri 2 build tooling.

Expected build command:

```bash
pnpm tauri build
```

## Installer

Preferred installer output:

- MSI or NSIS installer generated by Tauri
- Signed executable/installer when a code signing certificate is available

## App Data Location

Recommended app data path:

```text
%APPDATA%/StepForgeRecorder/
```

Subfolders:

```text
/data
/screenshots
/exports
/logs
```

## Configuration

MVP should use local SQLite settings.

Do not use JSON config files unless absolutely necessary.

## Updates

Automatic updates are not required for MVP.

Future versions may support:

- Tauri updater
- Signed releases
- Enterprise-controlled update channels

## Release Checklist

Before release:

- App builds successfully.
- Installer runs successfully.
- App opens after install.
- Recording works.
- Screenshots save correctly.
- DOCX export works.
- PDF export works.
- Settings persist.
- App uninstalls cleanly.
- Logs do not contain sensitive data.
- No development-only debug panels are visible.

## Versioning

Use semantic versioning:

```text
MAJOR.MINOR.PATCH
```

Example:

```text
0.1.0
```

MVP version should start below `1.0.0`.

## Build Artifacts

Do not commit generated installers to source control unless release process requires it.

Recommended ignored artifacts:

```text
/src-tauri/target
/dist
/node_modules
```

## Code Signing

Code signing is strongly recommended before distributing outside a trusted internal group.

Unsigned apps may trigger Windows SmartScreen warnings.

## Enterprise Deployment Future

Future enterprise deployment may require:

- Silent install
- MSI parameters
- Default settings policy
- Install for all users
- Centralized update control
- Group Policy support
